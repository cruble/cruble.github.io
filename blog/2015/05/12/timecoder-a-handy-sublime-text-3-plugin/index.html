TimeCoder - a Handy Sublime Text 3 Plugin

written

One of the cool things about the Flatiron School is that every lecture is recorded. That way, it is easy to go back over a topic that might still be fuzzy - or that you might want to review after gaining more perspective. But these lectures can be two hours long and cover a lot of ground. It can be hard to get right to a specific area or topic.

Our instructor, Steven Nunez has us all clap at the beginning of a lecture, for his own audio sync (and to wake us up). But it also marks the beginning of his video (or close to it). I thought my review sessions could be a lot more effecient if I had a timecode to match my notes. I write in Sublime and it turns out that it has a robust API and its pretty straightforward to get started with Sublime Text Plugin development. Forget about the fact that their language is based in Python and I have been spending the last 6+ weeks learning Ruby.

In the words of a famous programmer, “We’re programmers, damnit!” So I went for it.

I’ll go over the steps I took to create TimeCoder, my first Sublime Text 3 Plugin. But for a comprehensive tutorial, I recommend this one and this one as good starting points. You’ll also want to take a good look at the Sublime Text API Reference. You, too, can extend the functionality of your text editor with ease. Before I do that, here’s a short video showing the functionality of TimeCoder.

Sublime Makes it really easy to create a plug in. You go to Tools->New Plugin… and you get a new document with some python code to get you started.
