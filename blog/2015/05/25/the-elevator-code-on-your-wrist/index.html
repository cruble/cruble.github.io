The Elevator Code on Your Wrist

written

At the Flatiron School, we have been covering a lot of ground - from Javascript to JQuery, Rake to Rails. I can’t remember a time when I have been exposed to material of this breadth, depth, and usefulness. But I’m having a particularly hard time with remembering the five digit elevator code to get access to the second floor.

I was fine the first time, even the second round, which had some sequential numbers in it. But my brain seems to be too full to commit the latest five digit code to memory.

There are simple analog solutions, like writting the code on my hand or on a piece of paper. But why do that when it seemed like the perfect excuse to toy around with the Pebble Software Developer Kit.

Having spent a whopping five days learning Javascript, I felt expert enough to dive into PebbleJS, a Javascript based watch application tool (still in Beta) to see if I could code a solution to my Elevator Code conundrum.

I had a lot of fancy ideas about triggering a location or time-based reminder. Or even creating a JSON endpoint that could be updated and read remotely. But we are in project mode and I have a tech talk to prepare for next week. So I have put the fancy features in the icebox and opted for a six-line solution. Essentially, I am using Pebble’s UI library, creating a Card object and displaying it on the screen.

Now, of course, that’s not the real code, so don’t get any ideas about coming to Flatiron with it. The only way to get into the school is to apply.

Pebble raised over $10 million dollars in May of 2012 with their first Kickstarter campaign making the the largest and most successful campaign of the time. They did it again with their new watch, which raised $20,338,986 with over 74,000 backers in March of this year. The Pebble community is active and its core users are dedicated.

Also, the Pebble Developer tools are very easy to use. CloudPebble is an online console and development environment that also handles deploying to your watch - including an online watch emulator. And they have spent a lot of time making the API easy to use with lots of sample code.. For example, making an ajax call is fairly straightforward.

We don’t know how many Apple Watches have been sold, but its clear that the wearable market - whether its big or small - is not just a fad. As a programmers, its good to think of the small screen on the wrist as an extension of any application. Certainly, PebbleJS is a good place for anyone with Javascript experience to get started.

If your project team has time, you might want to think about creating a JSON endpoint to open up your data to the world of wearables.
